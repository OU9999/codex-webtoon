# Test Rules

## 기본 원칙

- 동작 변경은 가능하면 먼저 실패하는 테스트나 재현 절차를 만든 뒤 구현한다.
- 순서는 `재현/실패 확인 -> 구현 -> 통과 확인 -> 리팩터`를 따른다.
- 구조 변경과 동작 변경을 한 번에 섞지 않는다.
- 우선순위는 `동작 -> 정확성 -> 속도`다.

## 검증 명령

- 패키지 매니저는 항상 `pnpm`을 사용한다.
- 타입 변경이나 공유 모델 변경 후 `pnpm typecheck`를 실행한다.
- 서버, 상태 모델, 프롬프트, 데이터 변환 변경 후 `pnpm test`를 실행한다.
- 배포 결과물이나 빌드 설정 변경 후 `pnpm build`를 실행한다.
- 포맷 영향이 있는 문서/코드 변경 후 `pnpm format:check` 또는 범위 지정 Prettier 검증을 실행한다.

## UI 검증

- UI 동작 변경은 가능한 경우 실제 브라우저에서 확인한다.
- 레이아웃, 반응형, 시각 상태를 바꾸면 Playwright 스크린샷으로 겹침, 잘림, 빈 화면 여부를 확인한다.
- 디자인 변경은 `DESIGN.md`와 비교해 색상, 간격, 타이포그래피가 어긋나지 않는지 확인한다.
