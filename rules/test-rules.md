# Test Rules

## 기본 원칙

- 동작 변경 전 실패 테스트 또는 재현 절차 우선
- `재현/실패 확인 -> 구현 -> 통과 확인 -> 리팩터` 순서 준수
- 구조 변경과 동작 변경 혼합 금지
- `동작 -> 정확성 -> 속도` 우선순위

## 검증 명령

- 항상 `pnpm` 사용
- 타입 변경 또는 공유 모델 변경 후 `pnpm typecheck` 실행
- 서버, 상태 모델, 프롬프트, 데이터 변환 변경 후 `pnpm test` 실행
- 배포 결과물 또는 빌드 설정 변경 후 `pnpm build` 실행
- 포맷 영향이 있는 문서/코드 변경 후 `pnpm format:check` 또는 범위 지정 Prettier 검증

## UI 검증

- UI 동작 변경 시 실제 브라우저 확인
- 레이아웃, 반응형, 시각 상태 변경 시 Playwright 스크린샷 확인
- 겹침, 잘림, 빈 화면 여부 확인
- 디자인 변경 시 `DESIGN.md` 기준 색상, 간격, 타이포그래피 확인
