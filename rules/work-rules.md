# Work Rules

## 작업 전

- 요구사항 불명확 시 구현 전 질문
- 해석 복수 시 가능한 선택지 제시
- 요청 범위 밖 기능 추가 금지
- 관련 파일과 기존 패턴 확인 후 변경

## 코드 작성

- 요청에 필요한 최소 변경
- 기존 구조·네이밍·추상화 수준 우선
- 인접 코드 임의 개선·리팩터링 금지
- 내 변경이 만든 고아 코드만 제거
- 버그 원인 재현 후 자율 수정
- 해키한 우회 대신 구조 재검토

## 작업 단위

- 동작 변경과 구조 변경 분리
- 한 커밋 한 의도
- 관련 검증 통과 후 커밋
- 큰 작업의 작은 단계 분할
- 각 단계마다 검증 가능한 상태 유지

## Git

- 커밋 생성 시 `$commit` 스킬 사용
- PR 생성 시 `$pr` 스킬 사용
