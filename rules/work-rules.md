# Work Rules

## 작업 전

- 요구사항이 불명확하면 구현 전에 질문한다.
- 해석이 여러 개면 가능한 선택지를 제시하고 임의로 결정하지 않는다.
- 요청 범위를 벗어난 기능은 추가하지 않는다.
- 관련 파일과 기존 패턴을 먼저 확인한 뒤 변경한다.

## 코드 작성

- 변경 범위는 요청에 필요한 최소 범위로 제한한다.
- 기존 구조, 네이밍, 추상화 수준을 우선 따른다.
- 인접 코드를 임의로 개선하거나 리팩터링하지 않는다.
- 고아 코드 제거는 내 변경이 만든 것에 한정한다.
- 버그는 재현 가능한 원인을 찾고 자율적으로 수정한다.
- 해키한 우회가 필요하면 우회 대신 구조를 재검토한다.

## 작업 단위

- 동작 변경과 구조 변경은 가능하면 분리한다.
- 한 커밋은 한 의도만 담는다.
- 커밋은 관련 검증이 통과한 뒤에만 만든다.
- 큰 작업은 작은 단계로 나누고 각 단계마다 검증 가능한 상태를 만든다.
