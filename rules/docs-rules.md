# Docs Rules

## 작성 원칙

- 문서는 사용자가 실제로 판단하거나 실행하는 데 필요한 내용만 쓴다.
- 구현 세부보다 사용자에게 보이는 동작, 제약, 절차를 우선 설명한다.
- 같은 내용을 여러 문서에 중복해 쓰지 않는다. 필요하면 대표 문서로 연결한다.
- 한국어 문서와 영어 문서가 함께 있으면 내용 차이가 생기지 않게 맞춘다.

## 형식

- 제목과 목록은 짧고 구체적으로 쓴다.
- 명령어, 파일 경로, 환경 변수, 코드 식별자는 백틱으로 감싼다.
- 순서가 중요한 절차는 번호 목록을 사용한다.
- 긴 설명보다 작은 섹션으로 나누어 스캔 가능하게 작성한다.

## 수정 범위

- 문서 수정은 요청 범위와 직접 관련된 문서로 제한한다.
- 문서 구조를 바꿀 때는 링크와 참조 경로가 깨지지 않았는지 확인한다.
- 규칙 문서를 수정할 때는 `AGENTS.md`와 `CLAUDE.md`의 참조 경로도 함께 확인한다.
