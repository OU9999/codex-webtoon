# Docs Rules

스킬·에이전트·규칙 문서의 한국어 작성 규칙. 지시·목록형 내용은 간결한 명사구 기준.

## 대상

- `.claude`·`.codex`·`.agents` 하위 스킬·에이전트 파일
- `rules/` 하위 규칙 문서
- `AGENTS.md`, `CLAUDE.md`

## 제외

- `docs/` 하위 외부 공개 문서
- `README.md`, `README.kr.md` 등 사용자 대상 문서
- 서술이 필요한 본문 단락, 개념 설명
- 코드 블록, 인용, 예시 출력

## 원칙

- **서술형 종결 회피** — `~합니다`, `~입니다`, `~습니다`, `~한다`, `~하세요`, `~해야 한다` 같은 장황한 종결 어미 회피
- **명사구 우선** — 목록 항목은 동작·대상이 드러나는 명사구로 마무리
- **`~함` 남용 금지** — 동작 설명은 명사구 우선, 주체·조건이 필요한 원칙 문장에 한해 `~함` 허용
- **한 항목 한 내용** — 군더더기 수식어·중복 설명 제거
- **문서 내 일관성** — 같은 개념은 같은 용어로, 종결 방식도 문서 단위로 통일

## 교정 예시

| 서술형·장황                                         | 명사구·간결               |
| --------------------------------------------------- | ------------------------- |
| 이 스크립트는 빌드를 실행합니다.                    | 빌드 실행                 |
| URL 기준을 적용함.                                  | 공용 URL 기준 적용        |
| 테이블을 생성하고 데이터를 연결한다.                | 테이블 생성 + 데이터 연결 |
| 사용자가 앱을 명시하지 않았다면 먼저 질문해야 한다. | 앱 미지정 시 대상 질문    |

## 적용 시점

- 스킬·에이전트·규칙 문서 신규 작성 또는 수정
- 기존 문서는 해당 파일 수정 시 점진 교정
- 일괄 교정 강제 없음
