# 작업 정리

## 목표

흩어진 진행 기록과 현재 상태 기반으로 지금 할 만한 작업 3-5개 압축. 사용자 승인 후 가장 작은 단위부터 실행.

## 실행 순서

### 1. 현재 상태 확인

```bash
git status --short
git branch --show-current
```

dirty working tree 존재 시 진행 중 파일 우선 분류.

### 2. 기록 읽기

다음 순서로 필요한 만큼만 읽기.

1. 사용자가 지정한 `.progress/YYMMDD/*.md`
2. 최신 `.progress` 파일 5-10개
3. 관련 `.docs` 파일
4. `package.json`, README, 설정 파일처럼 현재 목표와 직접 관련된 파일

사용자가 제외한 작업은 별도 제외 목록으로 유지.

### 3. 작업 후보 만들기

각 후보는 다음 정보 포함.

- 작업명
- 왜 지금 필요한지
- 영향을 받는 파일/영역
- 검증 방법
- 위험도: 낮음/중간/높음

이미 완료된 작업, 대화 근거 없는 추측성 TODO, 막연한 리팩터링 제외.

### 4. 우선순위

기본 우선순위 기준:

1. 사용자가 바로 막혀 있는 버그/검증 실패
2. 릴리스/패키징을 막는 작업
3. 핵심 사용자 흐름의 UI/UX 문제
4. 문서/README/진행 기록 정리
5. 선택적 개선

### 5. 실행 전환

사용자가 "진행", "1번 작업", "좋아 진행"처럼 승인하면 선택 작업 즉시 시작. 범위 불명확 시 질문 하나만 사용.

## 보고 형식

```markdown
남은 작업:
- [우선순위] 작업명 — 이유 / 검증 방법

추천:
1. 지금 바로 할 작업
2. 그 다음 작업

제외:
- 사용자가 제외한 항목
```
