# 작업 정리

## 목표

흩어진 진행 기록과 현재 상태를 읽고 지금 할 만한 작업을 3-5개로 압축한다. 사용자가 진행을 승인하면 가장 작은 단위부터 실행한다.

## 실행 순서

### 1. 현재 상태 확인

```bash
git status --short
git branch --show-current
```

정리되지 않은 working tree가 있으면 어떤 파일이 진행 중인지 먼저 분류한다.

### 2. 기록 읽기

다음 순서로 필요한 만큼만 읽는다.

1. 사용자가 지정한 `.progress/YYMMDD/*.md`
2. 최신 `.progress` 파일 5-10개
3. 관련 `.docs` 파일
4. `package.json`, README, 설정 파일처럼 현재 목표와 직접 관련된 파일

사용자가 제외한 작업은 별도 제외 목록으로 유지한다.

### 3. 작업 후보 만들기

각 후보는 다음 정보를 포함한다.

- 작업명
- 왜 지금 필요한지
- 영향을 받는 파일/영역
- 검증 방법
- 위험도: 낮음/중간/높음

이미 완료된 작업, 대화에 근거가 없는 추측성 TODO, 막연한 리팩터링은 넣지 않는다.

### 4. 우선순위

기본 우선순위는 다음 기준으로 정한다.

1. 사용자가 바로 막혀 있는 버그/검증 실패
2. 릴리스/패키징을 막는 작업
3. 핵심 사용자 흐름의 UI/UX 문제
4. 문서/README/진행 기록 정리
5. 선택적 개선

### 5. 실행 전환

사용자가 "진행", "1번 작업", "좋아 진행"처럼 승인하면 선택된 작업을 바로 시작한다. 범위가 불명확하면 질문은 하나만 한다.

## 보고 형식

```markdown
남은 작업:
- [우선순위] 작업명 — 이유 / 검증 방법

추천:
1. 지금 바로 할 작업
2. 그 다음 작업

제외:
- 사용자가 제외한 항목
```
